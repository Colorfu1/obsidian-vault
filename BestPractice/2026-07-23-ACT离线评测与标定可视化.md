## Rerun 离线可视化

### Recorded images 与 calibrated agentview

左侧两路 recorded images 保留 LeRobot/OpenVLA 模型所见的内容和方向。右上角
`Physically calibrated agentview` 使用当前 LIBERO task 的 MuJoCo 固定相机外参和内参，
将机器人及动作几何投影到独立的物理相机图像副本。

当前 LeRobot LIBERO 数据采用固定的 180 度图像约定。对本地已验证的数据，渲染器的高度
翻转负责把 OpenGL buffer 转为正立图像，额外的宽度翻转使 stored image 相对物理 pinhole
投影呈水平反射。因此 calibrated entity 仅为可视化做一次水平反射；state、action、训练
数据、normalization 和 ACT 输入保持不变。

离线命令必须显式声明已验证的约定：

```bash
scripts/experiments/act/eval_offline.sh \
  --policy-path=outputs/act/a_sanity/<run>/checkpoints/<step>/pretrained_model \
  --suite=libero_object \
  --task-id=8 \
  --dataset-root=datasets/lerobot/lerobot/libero \
  --agentview-image-convention=lerobot_180 \
  --episodes 823 826 834 840 841 \
  --num-visualized-episodes=1 \
  --output-dir=outputs/act/a_sanity/<run>/offline_5
```

默认 `auto` 只接受带明确 image-convention metadata 的 generated cache；显式 dataset root
不会仅凭目录名猜测方向。该参数只影响 calibrated visualization。

### Panda 骨架与动作语义

- 蓝色 Panda 骨架由 recorded 8D EEF state 通过连续 IK 重建，中间关节不是数据集实测
  joint state；IK 失败时明确采用 stale/omitted 语义。
- 深蓝线是 recorded EEF 历史。
- 每个 recorded EEF 上的浅蓝短线是 GT 首动作 OSC position command。
- 同一起点的深橙短线是 teacher-forced Pred 首动作，未在模拟器执行。
- 半透明橙线是当前帧完整 future chunk 的累计 command preview，同样未执行。
- 所有机器人几何共享以 base 为原点的显示坐标系，并标注 `X`、`Y Forward`、`Z Up`。

### Action 与 metric 展示

下方 action chart 提供 14 条可独立点选的曲线，七个维度分别将 GT 与 Pred 相邻排列并使用
不同颜色。Rerun 会按当前可见曲线自动缩放纵轴，因此视觉间距必须结合数值指标判断。

`Offline signals -> Metrics` 已改为两张紧凑表格：第一张展示总体 offline metrics，第二张
展示 `x/y/z/roll/pitch/yaw/gripper` 七维 first-action MAE。完整 horizon 数据仍保留在
`metrics.json`，Rerun 只提供便于快速查看的摘要。
